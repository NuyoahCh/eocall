项目介绍：基于 Go 语言开发的 OnCall 响应引擎，构建集知识库、对话与运维于一体，通过 AI Agent 实现故障秒级应答与根因分析

个人职责：

工具链编排： 设计标准化工具协议，封装日志监控查询能力，实现 LLM 意图识别、参数解析与自主编排的完整链路
知识库检索增强： 搭建 RAG 检索系统，基于向量数据库与语义匹配技术，实现运维文档的高精度智能问答与知识召回
智能体决策闭环：基于 Plan-Execute 模式构建运维 Agent，打通从告警分析、方案规划到工具执行的自动化闭环

技术难点：

长上下文优化： 设计滑动窗口摘要机制，保留最近 50 轮对话并压缩历史，在保障连贯性前提下降低 60% Token 开销。
并发会话隔离： 构建基于 UserID 的会话隔离容器，为每个请求分配独立状态实例，彻底解决百级并发下的上下文混淆问题

流式响应重构：重构推理响应链路，采用 Streaming 流式输出实现 LLM 逐字推送，消除大模型生成延迟对体验的影响

使用golang作为主语言技术栈，同时使用ByteDance的Eino大语言模型应用框架，中间件根据实际的场景进行选择
